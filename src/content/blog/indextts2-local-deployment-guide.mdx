## 引言

2025 年 9 月，Bilibili 开源了 **IndexTTS2**——首个支持情感可控、时长精确调节的自回归零样本语音合成模型。它可以用一段几秒的参考音频克隆任意音色，还能独立控制说话的情感和语气。

想象一下这个场景：

> 你给模型一段 5 秒的平静人声作为音色参考，再给它一句"快躲起来！他要来了！"的文本，加上一段恐惧风格的情感参考——模型就能用**克隆的音色**，带着**恐惧的语气**说出这句话，效果相当逼真。

本文记录我在 Windows 上完整部署 IndexTTS2 的过程，包含所有步骤、命令和踩坑解决方案。

---

## 一、IndexTTS2 是什么

IndexTTS2 的核心能力：

| 能力 | 说明 |
|------|------|
| **零样本音色克隆** | 用几秒参考音频即可克隆任意人声 |
| **情感解耦控制** | 音色和情感可独立指定 |
| **时长控制** | 可精确控制合成语音的时长（v2 新增） |
| **多情感维度** | 支持高兴、愤怒、悲伤等 8 种情感 |
| **文本情感引导** | 直接用文字描述来控制情感 |

> 论文地址：[arXiv:2506.21619](https://arxiv.org/abs/2506.21619)

## 二、环境准备

### 2.1 项目信息

```bash
# GitHub 仓库
https://github.com/index-tts/index-tts

# 克隆项目
git clone https://github.com/index-tts/index-tts.git
cd index-tts
```

### 2.2 安装 uv 包管理器

IndexTTS2 项目**只支持** `uv` 作为包管理器。官方明确说了用 pip/conda 导致的问题不予受理。

```bash
# 用 pip 安装 uv（如果你的机器已有 Python）
pip install -U uv

# 验证
uv --version
# 输出示例：uv 0.11.20
```

关于 uv 的理解：

- **uv 命令**是全局安装的，装一次所有项目可用
- **虚拟环境**（`.venv`）是每个项目独立的，`uv sync` 时会自动创建
- uv 会自动根据项目的 `.python-version`（本项目要求 3.10）下载对应 Python 版本到 `.venv` 中

### 2.3 安装项目依赖

```bash
cd index-tts

# 安装基础依赖 + WebUI 支持（推荐）
uv sync --extra webui
```

如果下载慢，使用国内 PyPI 镜像：

```bash
uv sync --extra webui --default-index "https://mirrors.aliyun.com/pypi/simple"
```

> 💡 **关于 DeepSpeed**：Windows 用户建议去掉 `--all-extras` 只加 `--extra webui`，因为 DeepSpeed 在 Windows 上编译容易出问题。

这一步会做三件事：

1. 下载 Python 3.10 到项目 `.venv`
2. 安装 `pyproject.toml` 中声明的所有依赖
3. 生成 `uv.lock` 锁定文件

### 2.4 我的环境

| 项目 | 版本/配置 |
|------|----------|
| 系统 | Windows |
| Python（系统） | 3.13.5 |
| Python（项目） | 3.10（uv 自动管理） |
| uv | 0.11.20 |
| GPU | 可选（无 GPU 也能跑，但慢） |

## 三、下载模型权重

`checkpoints/` 目录里只有配置文件，需要手动下载完整模型。模型文件合计约 **4.7 GB**。

### 3.1 方式一：ModelScope（国内推荐）

```bash
# 安装 modelscope 命令行工具
uv tool install "modelscope"

# 下载模型
modelscope download --model IndexTeam/IndexTTS-2 --local_dir checkpoints
```

> ⚠️ `uv tool install` 安装的命令行工具不在系统的 PATH 中，直接用 `modelscope` 命令会提示找不到。需要用 `uv tool run` 或 `uvx` 来执行：
>
> ```bash
> uvx modelscope download --model IndexTeam/IndexTTS-2 --local_dir checkpoints
> ```

### 3.2 方式二：HuggingFace（需要科学上网）

```bash
uv tool install "huggingface-hub[cli,hf_xet]"
hf download IndexTeam/IndexTTS-2 --local-dir=checkpoints
```

### 3.3 下载后的目录结构

```
checkpoints/
├── config.yaml             ✅ 已有（项目自带的配置）
├── pinyin.vocab            ✅ 已有（拼音对照表）
├── gpt.pth                 ⬇️ 3.48 GB（核心 GPT 模型）
├── s2mel.pth               ⬇️ 1.20 GB（声码器）
├── feat1.pt                ⬇️ 57 KB（说话人特征矩阵）
├── feat2.pt                ⬇️ 375 KB（情感特征矩阵）
└── wav2vec2bert_stats.pt   ⬇️ 9 KB（音频统计信息）
```

## 四、GPU 检测（可选但推荐）

运行项目自带的检测脚本确认 PyTorch 是否能识别 GPU：

```bash
uv run tools/gpu_check.py
```

如果输出中能看到 CUDA 和你的显卡型号，说明 GPU 加速可用。如果没有 GPU，项目会 fallback 到 CPU，推理速度会慢很多。

## 五、启动 WebUI

### 5.1 首次启动 — 注意 HuggingFace 网络问题

```bash
# 启动 WebUI（cmd 环境）
uv run webui.py

# 浏览器访问 http://127.0.0.1:7860
```

**首次启动会遇到的问题**：GPT 主模型加载成功（`GPT weights restored from: ./checkpoints/gpt.pth` ✅），但代码会自动从 HuggingFace 下载 `facebook/w2v-bert-2.0` 这个辅助模型来处理音频特征。在中国大陆访问 HuggingFace 超时会导致启动失败。

### 5.2 ✅ 解决方案：使用 HuggingFace 镜像

**cmd 环境：**

```cmd
set HF_ENDPOINT=https://hf-mirror.com
uv run webui.py
```

**PowerShell 环境：**

```powershell
$env:HF_ENDPOINT="https://hf-mirror.com"
uv run webui.py
```

`w2v-bert-2.0` 是一个很小的模型（几百 KB），使用镜像下载一次后就会缓存到本地，之后再启动就不需要联网了。

### 5.3 节省显存：FP16 模式

```bash
uv run webui.py --fp16
```

FP16 半精度推理可以显著降低显存占用，质量损失很小，推荐开启。

### 5.4 更多参数

```bash
# 查看所有参数
uv run webui.py -h
```

## 六、Python 脚本调用（进阶）

除了 WebUI，IndexTTS2 也支持直接通过 Python 代码调用，适合批量处理或集成到其他应用。

### 6.1 基本语音合成（音色克隆）

```python
from indextts.infer_v2 import IndexTTS2

tts = IndexTTS2(
    cfg_path="checkpoints/config.yaml",
    model_dir="checkpoints",
    use_fp16=False
)

text = "欢迎来到我的博客，今天我们来聊聊语音合成。"
tts.infer(
    spk_audio_prompt='examples/voice_01.wav',
    text=text,
    output_path="output.wav",
    verbose=True
)
```

运行方式：

```bash
PYTHONPATH="$PYTHONPATH:." uv run your_script.py
```

### 6.2 带情感控制的合成

```python
# 用情感参考音频来控制语气
tts.infer(
    spk_audio_prompt='examples/voice_07.wav',
    text="酒楼丧尽天良，开始借机竞拍房间！",
    output_path="output_emo.wav",
    emo_audio_prompt="examples/emo_sad.wav",
    verbose=True
)
```

### 6.3 8 维情感向量控制

情感向量顺序：`[高兴, 愤怒, 悲伤, 害怕, 厌恶, 忧郁, 惊讶, 平静]`

```python
# 悲伤语气（0.8 权重）
tts.infer(
    spk_audio_prompt='examples/voice_07.wav',
    text="真遗憾，我们失败了。",
    output_path="output_vector.wav",
    emo_vector=[0, 0, 0.8, 0, 0, 0, 0, 0],
    verbose=True
)
```

### 6.4 文本自动情感引导

```python
# 根据文本内容自动生成情感
tts.infer(
    spk_audio_prompt='examples/voice_12.wav',
    text="快躲起来！是他要来了！他要来抓我们了！",
    output_path="output_text_emo.wav",
    emo_alpha=0.6,
    use_emo_text=True,
    verbose=True
)
```

## 七、踩坑记录

### 7.1 HuggingFace 连接超时

**现象**：启动时报 `ConnectTimeoutError: Connection to huggingface.co timed out`。

**原因**：首次运行需要下载 `facebook/w2v-bert-2.0` 辅助模型，中国大陆网络无法直连 HuggingFace。

**解决**：

```powershell
# PowerShell
$env:HF_ENDPOINT="https://hf-mirror.com"

# 或 CMD
set HF_ENDPOINT=https://hf-mirror.com
```

### 7.2 `modelscope` 命令找不到

**现象**：安装 modelscope 后执行命令报 `无法将"modelscope"项识别为 cmdlet`。

**原因**：`uv tool install` 安装的工具在 uv 自己的目录中，默认不在系统 PATH 里。

**解决**：

```bash
# 方法一：用 uvx 运行
uvx modelscope download --model IndexTeam/IndexTTS-2 --local_dir checkpoints

# 方法二：查看 uv 的工具目录并加入 PATH
uv tool dir
# 输出类似：C:\Users\用户名\.local\bin
```

### 7.3 DeepSpeed 编译失败（Windows）

**现象**：`uv sync --all-extras` 时报 DeepSpeed 编译错误。

**原因**：DeepSpeed 在 Windows 上编译比较麻烦，需要 Visual Studio Build Tools 和 CUDA Toolkit。

**解决**：去掉 DeepSpeed，只装 WebUI：

```bash
# 重新同步，不装 DeepSpeed
uv sync --extra webui
```

### 7.4 磁盘空间不足

**现象**：下载模型时提示磁盘空间不足。

**原因**：模型文件合计约 4.7 GB，加上依赖和 Python 运行环境共需约 8~10 GB。

**解决**：清理磁盘或检查 `checkpoints/` 是否已有未删干净的旧文件。

## 八、完整启动流程一览

```bash
# 1. 安装 uv 包管理器
pip install -U uv

# 2. 安装项目依赖
cd D:\path\to\index-tts
uv sync --extra webui --default-index "https://mirrors.aliyun.com/pypi/simple"

# 3. 下载模型（并行进行，不依赖上一步完成）
uvx modelscope download --model IndexTeam/IndexTTS-2 --local_dir checkpoints

# 4. 启动 WebUI
$env:HF_ENDPOINT="https://hf-mirror.com"   # PowerShell 设置镜像
uv run webui.py --fp16                      # 开启 FP16 省显存

# 5. 浏览器打开 http://127.0.0.1:7860
```

## 九、总结

IndexTTS2 的本地部署整体流程清晰，主要门槛集中在两点：

1. **网络问题**：模型下载和辅助模型加载都依赖海外源，国内用户需要配置镜像
2. **依赖管理**：官方只支持 `uv`，不支持 pip/conda，需要额外安装

一旦环境配好，体验相当不错：

- ✅ WebUI 开箱即用，不需要写代码就能体验音色克隆
- ✅ 情感控制效果明显，说话的语气能真实传达出来
- ✅ Python API 简洁，方便二次开发

对于想入门 AI 语音合成的开发者来说，IndexTTS2 是一个很好的学习项目——代码结构清晰，用到了 GPT、VQ-VAE、扩散模型等多种主流技术。
