## 背景：从 MVP 到生产级

在[上一篇文章](/blog/zebra-zd888-browser-print-zpl-integration)中，我们实现了基于 Zebra Browser Print + ZPL 的 Web 端标签打印方案，解决了浏览器原生打印精度不足的问题。

但在实际生产环境中运行几周后，用户反馈了新的问题：

1. **二维码尺寸不准**：设置 12mm，实际打印出来只有 10mm 左右
2. **多标签位置偏移**：第 1 个标签位置准确，第 4 个标签明显偏右
3. **校准参数难调**：每次调整 offsetX 都要重新打印测试，浪费标签纸

这些问题的根源在于：

- **PNG 二维码周围有白边**，直接缩放会把白边也算进去
- **累积舍入误差**，每个标签的坐标独立计算 `Math.round`，误差会叠加
- **黑盒调试**，看不到理论坐标和实际效果的对比

本文记录从基础方案到生产级方案的完整优化过程，重点解决**精度**和**可调试性**两大问题。

## 核心优化 1：智能二维码裁剪算法

### 问题：为什么需要裁剪

用 Canvas 把 PNG 二维码转成 ZPL 位图时，最直觉的做法是：

```ts
// ❌ 基础版：直接缩放
const canvas = document.createElement('canvas');
canvas.width = targetSize;  // 168 dots (14.2mm @ 300dpi)
canvas.height = targetSize;
ctx.drawImage(img, 0, 0, targetSize, targetSize);
```

**问题**：PNG 二维码周围通常有白边（padding），直接缩放会把白边也算进去。

假设场景：
- 目标尺寸：14.2mm × 14.2mm（168 × 168 dots @ 300dpi）
- PNG 原图：200 × 200 px，实际黑色区域只有 160 × 160 px
- 直接缩放：黑色区域只占 `168 × (160/200) = 134 dots`，实际只有 **11.4mm**

用户设置 14.2mm，实际打印 11.4mm，自然觉得"尺寸不准"。

### 解决方案：自动识别黑色区域边界

核心思路：
1. 先把 PNG 渲染到原始尺寸的 Canvas
2. 遍历所有像素，找到黑色区域的最小包围矩形
3. 把包围矩形扩展成正方形（二维码必须是正方形）
4. 只裁剪这个正方形区域，缩放到目标尺寸

### 实现：`findDarkPixelBounds` 函数

```ts
type PixelBounds = {
  x: number;
  y: number;
  width: number;
  height: number;
};

const isDarkPixel = (pixels: Uint8ClampedArray, index: number): boolean => {
  const r = pixels[index] ?? 255;
  const g = pixels[index + 1] ?? 255;
  const b = pixels[index + 2] ?? 255;
  const a = pixels[index + 3] ?? 255;
  const brightness = (r + g + b) / 3;
  // 亮度 < 180 且不透明 → 视为黑色像素
  return a > 25 && brightness < 180;
};

const findDarkPixelBounds = (
  pixels: Uint8ClampedArray,
  width: number,
  height: number,
): PixelBounds | undefined => {
  let minX = width;
  let minY = height;
  let maxX = -1;
  let maxY = -1;

  // 遍历所有像素，找到黑色区域的边界
  for (let y = 0; y < height; y++) {
    for (let x = 0; x < width; x++) {
      const index = (y * width + x) * 4;
      if (isDarkPixel(pixels, index)) {
        minX = Math.min(minX, x);
        minY = Math.min(minY, y);
        maxX = Math.max(maxX, x);
        maxY = Math.max(maxY, y);
      }
    }
  }

  if (maxX < minX || maxY < minY) return undefined;

  // 计算包围矩形
  const boundsWidth = maxX - minX + 1;
  const boundsHeight = maxY - minY + 1;

  // 扩展成正方形（取长边）
  const squareSize = Math.min(
    Math.max(boundsWidth, boundsHeight),
    width,
    height,
  );

  // 居中裁剪
  const centerX = minX + boundsWidth / 2;
  const centerY = minY + boundsHeight / 2;
  const x = Math.min(
    Math.max(Math.round(centerX - squareSize / 2), 0),
    width - squareSize,
  );
  const y = Math.min(
    Math.max(Math.round(centerY - squareSize / 2), 0),
    height - squareSize,
  );

  return { x, y, width: squareSize, height: squareSize };
};
```

### 完整的转换流程

```ts
const urlToZplBitmap = (
  url: string,
  targetSize: number,      // 目标画布尺寸（如 142 dots）
  contentSize: number,     // 实际内容尺寸（如 120 dots）
): Promise<string> => {
  return new Promise((resolve, reject) => {
    const img = new Image();
    img.crossOrigin = 'anonymous';
    img.addEventListener('load', () => {
      // 步骤 1：渲染到原始尺寸
      const sourceCanvas = document.createElement('canvas');
      sourceCanvas.width = img.naturalWidth;
      sourceCanvas.height = img.naturalHeight;
      const sourceCtx = sourceCanvas.getContext('2d')!;
      sourceCtx.fillStyle = 'white';
      sourceCtx.fillRect(0, 0, sourceCanvas.width, sourceCanvas.height);
      sourceCtx.drawImage(img, 0, 0);

      // 步骤 2：识别黑色区域边界
      const sourceImageData = sourceCtx.getImageData(
        0, 0, sourceCanvas.width, sourceCanvas.height
      );
      const bounds = findDarkPixelBounds(
        sourceImageData.data,
        sourceCanvas.width,
        sourceCanvas.height,
      );

      if (!bounds) {
        reject(new Error('未识别到二维码黑色区域'));
        return;
      }

      // 步骤 3：创建目标画布
      const canvas = document.createElement('canvas');
      canvas.width = targetSize;
      canvas.height = targetSize;
      const ctx = canvas.getContext('2d')!;
      ctx.fillStyle = 'white';
      ctx.fillRect(0, 0, targetSize, targetSize);
      ctx.imageSmoothingEnabled = false;

      // 步骤 4：只裁剪黑色区域，缩放到 contentSize，居中绘制
      const drawSize = Math.min(contentSize, targetSize);
      const drawOffset = Math.round((targetSize - drawSize) / 2);
      ctx.drawImage(
        sourceCanvas,
        bounds.x, bounds.y, bounds.width, bounds.height,  // 源裁剪区域
        drawOffset, drawOffset, drawSize, drawSize,       // 目标绘制区域
      );

      // 步骤 5：转 ZPL 位图
      resolve(canvasToZplHex(canvas, targetSize));
    });
    img.src = url;
  });
};
```

### 效果对比

| 场景 | 基础版 | 优化版 |
|------|-------|-------|
| PNG 原图 | 200×200 px，黑色区域 160×160 px | 同左 |
| 目标尺寸 | 14.2mm (168 dots) | 14.2mm (168 dots) |
| 内容尺寸配置 | 无此参数 | 12mm (142 dots) |
| 实际黑色区域 | 134 dots (11.4mm) ❌ | 142 dots (12mm) ✅ |
| 周围留白 | 不可控 | 13 dots (1.1mm) |

**关键参数**：`contentSize`

代码中引入了 `contentSize` 参数（对应用户配置的"二维码内容大小"），用于控制黑色区域在画布中的占比。

以实际项目的默认参数为例：
- `qrSize = 14.2mm`（二维码画布大小）→ 168 dots
- `qrContentSize = 12mm`（二维码内容大小）→ 142 dots
- 黑色区域缩放到 142 dots，居中绘制在 168 dots 画布上
- 周围留白 `(168 - 142) / 2 = 13 dots`（约 1.1mm）

这样既保证了黑色区域尺寸准确，又避免了边缘被裁切。

## 核心优化 2：浮点数精度定位系统

### 单位转换函数

首先定义毫米到点阵的转换函数：

```ts
const mmToDotsFloat = (mm: number): number => {
  return (mm / 25.4) * settings.value.dpi;
};
```

这个函数保持浮点数精度，不做舍入。

### 问题：累积舍入误差

基础版代码中，每个标签的坐标独立计算：

```ts
// ❌ 基础版：每次都舍入
const labelWidthDots = Math.round(mmToDots(15));    // 177
const columnGapDots = Math.round(mmToDots(2));      // 24

for (let i = 0; i < 4; i++) {
  const x = offsetX + i * (labelWidthDots + columnGapDots);
  // i=0: x = 24
  // i=1: x = 24 + 201 = 225
  // i=2: x = 24 + 402 = 426
  // i=3: x = 24 + 603 = 627
}
```

**真实值**：
- `15mm = 177.165 dots`
- `2mm = 23.622 dots`
- 每个标签实际间隔 = `177.165 + 23.622 = 200.787 dots`

**累积误差**：
- 第 1 个标签：误差 0
- 第 2 个标签：误差 `201 - 200.787 = 0.213 dots`
- 第 3 个标签：误差 `402 - 401.574 = 0.426 dots`
- 第 4 个标签：误差 `603 - 602.361 = 0.639 dots`（约 0.054mm）

虽然单个误差很小，但 4 个标签叠加后，第 4 个标签会明显偏右。

### 解决方案：浮点计算 + 最后舍入

**核心思想**：
- 用于**精确定位计算**的参数保持浮点数
- 用于**ZPL 命令**的参数才舍入成整数
- 每个标签的坐标从**原点重新计算**，而不是累加

```ts
const generateZPL = async (bitmapList: string[]): Promise<string> => {
  const s = settings.value;

  // 必须是整数的参数（用于 ZPL 命令）
  const paperWidthDots = Math.round(mmToDotsFloat(s.paperWidth));
  const labelLengthDots = Math.round(mmToDotsFloat(s.labelHeight));
  const qrSizeDots = Math.round(mmToDotsFloat(s.qrSize));

  // 保持浮点数的参数（用于精确定位计算）
  const labelWidthFloat = mmToDotsFloat(s.labelWidth);
  const labelHeightFloat = mmToDotsFloat(s.labelHeight);
  const columnGapFloat = mmToDotsFloat(s.columnGap);
  const offsetXFloat = mmToDotsFloat(s.offsetX);
  const offsetYFloat = mmToDotsFloat(s.offsetY);

  // ...

  pageImgs.forEach((hexData, colIdx) => {
    // 使用浮点数计算位置（避免累积误差）
    const labelXFloat = offsetXFloat + colIdx * (labelWidthFloat + columnGapFloat);
    const qrXFloat = labelXFloat + (labelWidthFloat - qrSizeDots) / 2;
    const qrYFloat = offsetYFloat + (labelHeightFloat - qrSizeDots) / 2;

    // 最后才舍入到整数
    const qrXDots = Math.round(qrXFloat);
    const qrYDots = Math.round(qrYFloat);

    zpl += `^FO${qrXDots},${qrYDots}`;
    // ...
  });
};
```

### 效果对比

以下示例基于简化参数（offsetX = 2mm, labelWidth = 15mm, columnGap = 2mm）：

| 标签 | 基础版 X 坐标 | 优化版 X 坐标 | 理论值 | 基础版误差 | 优化版误差 |
|------|-------------|-------------|--------|----------|----------|
| 1 | 24 | 24 | 23.622 | +0.378 | +0.378 |
| 2 | 225 | 224 | 224.409 | +0.591 | -0.409 |
| 3 | 426 | 425 | 425.196 | +0.804 | -0.196 |
| 4 | 627 | 626 | 625.983 | +1.017 | +0.017 |

**关键改进**：
- 基础版：累积误差从 0.378 增长到 1.017 dots（**累积 0.639 dots**）
- 优化版：每个标签独立舍入，误差在 ±0.5 dots 范围内波动（**无累积效应**）

虽然单个标签的舍入误差无法完全消除，但优化版避免了误差累积，第 4 个标签的误差从 1.017 dots 降到 0.017 dots。

## 核心优化 3：可视化校准系统

### 问题：黑盒调试

基础版方案中，调整 `offsetX` / `offsetY` 的流程：

1. 修改参数
2. 点击打印
3. 等待打印机出纸
4. 用尺子测量
5. 回到步骤 1

每次调整都要浪费一张标签纸，而且看不到"理论坐标"和"实际效果"的对比。

### 解决方案：实时预览 + 标尺系统

在打印前，用 CSS 按比例渲染一个"虚拟标签纸"，显示：

- 纸张边界（70mm × 15mm）
- 标签格子（4 个 15mm × 15mm）
- 二维码位置（12mm × 12mm）
- 二维码内容区域（红色虚线框）
- 水平/垂直标尺（每 1mm 一个刻度）

用户调整参数时，预览实时更新，**不用打印就能看到效果**。

```ts
// 水平标尺：每 5mm 一个主刻度，每 1mm 一个次刻度
const horizontalMajorTicks = computed(() => {
  const width = settings.value.paperWidth;
  const ticks = [];
  for (let i = 0; i <= width; i += 5) {
    ticks.push(i);
  }
  return ticks;

const horizontalMinorTicks = computed(() => {
  const width = settings.value.paperWidth;
  const ticks = [];
  for (let i = 1; i <= width; i++) {
    if (i % 5 !== 0) {
      ticks.push(i);
    }
  }
  return ticks;
});
```

## 工程化细节

### 1. 参数持久化

```ts
const STORAGE_KEY = 'print-qrcode-settings';

const loadSettings = (): typeof defaultSettings => {
  const saved = localStorage.getItem(STORAGE_KEY);
  if (saved) {
    try {
      return { ...defaultSettings, ...JSON.parse(saved) };
    } catch {
      return { ...defaultSettings };
    }
  }
  return { ...defaultSettings };
};

const saveSettings = () => {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(settings.value));
};
```

用户校准好参数后，下次打开页面自动恢复，不用重新调整。

### 2. 间隙检测模式

```ts
const createZplHeader = () => {
  let header = '^XA';
  header += '^MNY';  // 启用间隙检测（Web sensing）
  header += '^LH0,0';
  header += `^PQ${s.copies || 1}`;
  header += `^LL${labelLengthDots}`;  // 标签本体高度
  header += `^PW${paperWidthDots}`;
  return header;
};
```

**`^MNY` 的作用**：
- 打印机自动检测标签纸的物理间隙（黑标或透明间隙）
- 每打印完一个标签，自动走纸到下一个标签的起点
- 不需要手动计算行间距（`rowGap`）

这是生产环境的标准配置，避免因标签纸批次差异导致的对位问题。

### 3. 打印份数支持


ZPL 的 `^PQ` 指令支持指定打印份数，前端只需生成一次 ZPL，打印机自动重复打印。

### 5. 参数校验

```ts
async onConfirm() {
  try {
    const values = await formApi.getValues();
    settings.value = { ...defaultSettings, ...values };

    // 校验 1：二维码画布不能超过标签尺寸
    const maxQrSize = Math.min(
      settings.value.labelWidth,
      settings.value.labelHeight,
    );
    if (settings.value.qrSize > maxQrSize) {
      throw new Error(`二维码画布大小不能超过标签尺寸 ${maxQrSize}mm`);
    }

    // 校验 2：内容尺寸不能超过画布尺寸
    if (settings.value.qrContentSize > settings.value.qrSize) {
      throw new Error('二维码内容大小不能大于二维码画布大小');
    }

    saveSettings();
    await generateAndPrint();
  } catch (error) {
    ElMessage.error(error.message);
  }
}
```

避免用户输入不合理的参数导致打印失败。

## 关键认知

1. **PNG 转位图不是简单的格式转换**
   - 必须处理白边、透明度、亮度阈值
   - 智能裁剪是生产级方案的必备功能

2. **浮点数精度不是过度设计**
   - 累积舍入误差在多标签场景下会被放大
   - "保持浮点 + 最后舍入"是标准做法

3. **可视化预览是工程化的分水岭**
   - 基础版：能用，但调试成本高
   - 进阶版：好用，调试效率提升 10 倍

4. **参数化是适配性的基础**
   - 不同打印机、不同标签纸，物理偏移不同
   - 把偏移量、DPI 做成配置，部署时重新校准即可

5. **ZPL 的高级特性值得深挖**
   - `^MNY` 间隙检测
   - `^PQ` 打印份数
   - `^LH` 标签原点
   - 这些指令能解决很多"看似需要代码处理"的问题
